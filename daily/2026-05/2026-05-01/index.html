GitHub承认：AI编程已经把平台推向30倍规模，可靠性成核心卖点

GitHub CTO Vlad Fedorov 在4月28日公开承认，平台在2025年10月启动10倍扩容计划后，2026年2月已意识到需要按"今天30倍规模"重新设计。原因很直接：agentic development工作流加速，仓库创建、PR活跃度、API使用量、自动化任务全面攀升。

GitHub同时披露了近期两起事故：4月23日merge queue问题影响658个仓库、2092个PR；4月27日搜索故障让部分PR/Issue/Projects体验受阻。官方已将"可用性优先"摆在新功能前面。

🔐 安全与平台

OpenAI推出Advanced Account Security：ChatGPT和Codex账号安全门槛全面抬高

从4月30日起，启用该功能后要求passkey或物理安全密钥，关闭基于密码的登录；禁用邮箱和短信找回，改为备用passkey、安全密钥和recovery key；缩短登录会话、开启登录提醒，"对话不用于模型训练"自动开启。

2026年6月1日起，Trusted Access for Cyber个人成员将被强制要求启用该保护。随着用户在ChatGPT和Codex里处理代码、调试、研究材料甚至内部文档，账号本身已成高价值入口。

GitHub Copilot in Visual Studio4月更新：IDE从"补全插件"变成"任务调度器"

新版本支持直接在IDE里启动cloud agent session，让云端agent根据任务自动创建GitHub issue和Pull Request；自定义agents支持用户级定义存放在%USERPROFILE%/.github/agents/；Debugger agent可从GitHub/Azure DevOps issue出发，在真实运行时环境里复现和诊断问题。

本地IDE正在变成远程agent orchestration的前台，Visual Studio这次更新的方向很清晰。

🤖 AI模型与工具

NVIDIA Nemotron 3 Nano Omni：多模态感知agent模型，吞吐量提升9倍

NVIDIA发布开源多模态模型，支持文本、图像、音频、视频、文档、图表和图形界面输入。定位是作为agent系统里的perception sub-agent，与更强的规划模型协作，把视觉、语音和文本理解的感知层任务做成可工业化复用的组件。

未来agent系统大概率是"规划模型+感知模型+执行工具"的分层协作，而非一个超大模型包打天下。
